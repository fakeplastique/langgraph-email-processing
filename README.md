# Email Processor

Проект для автоматизованої обробки email-повідомлень. Використовує **LangGraph** та **LangChain** (з моделями Anthropic Claude) для класифікації та створення короткого змісту (summary) вхідних листів. 

Агенти обробляють повідомлення паралельно через граф (завантаження тіла листа -> класифікація та створення summary). В якості інфраструктури використовуються **Kafka**, **PostgreSQL** та локальне сховище для blob-даних.

## Архітектура та компоненти
* **Kafka** - брокер повідомлень для черги листів.
* **PostgreSQL** - реляційна база даних для збереження інформації та стану обробки.
* **Blob Store** - директорія (сховище) для сирих текстів (тіла) листів.
* **Email Processor** - основні воркери, що читають повідомлення і пропускають через LLM-граф. Можуть масштабуватися (за замовчуванням 3 репліки).
* **Dead Letter Replay** сервіс - для повторної обробки повідомлень, які не вдалося обробити з першого разу.

## Що покращити для prod-версії

* Залежно від кількості повідомлень, потрібно масштабувати кількість воркерів як варіант горизонтального масштабування (та збільшувати кількість partition в topic). При досягенні межі можливостей горизонтального масштабування, потрібно здійснити перехід до асинхронного event loop для читання повідомлень з Kafka та запису в PostgreSQL.
* Додати трейсинг помилок та метрики виконання запитів для оптимізації ботлнеків.
* Залежно від кількості повідомлень, можна оптимізовувати запис в БД через батчинг.
* Додати фолбек для використання іншого LLM-провайдера у разі недоступності основного (оскільки Anthropic API та OpenAI API можуть бути нестабільними).

## Налаштування та запуск (через Docker)

Проект налаштовано для швидкого запуску за допомогою `docker-compose`.

1. **Налаштування змінних оточення**
   Скопіюйте приклад конфігураційного файлу:
   ```bash
   cp .env.example .env
   ```
   Відкрийте створений файл `.env` та **обов'язково** додайте ваш ключ Anthropic API:
   ```dotenv
   APP_ANTHROPIC_API_KEY=ваш_ключ_тут
   ```

2. **Запуск проекту**
   ```bash
   docker-compose up --build -d
   ```

3. **Перевірка роботи**
   Після успішного запуску ви можете перевірити логи додатку:
   ```bash
   docker-compose logs -f email-processor
   ```

## Локальна розробка

Для запуску без Docker або розробки локально використовується (залежності керуються через `pyproject.toml`):

```bash
pip install -e .[dev]
```
